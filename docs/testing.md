# Стратегия тестирования KIBAD

## Обзор

KIBAD использует многоуровневый подход к тестированию, обеспечивающий корректность трансформаций данных, статистических вычислений и рабочих процессов интерфейса. Текущий объём: **343 теста** для всех модулей ядра.

## Категории тестов

### 1. Модульные тесты

Каждый модуль `core/` имеет соответствующий файл тестов. Модульные тесты проверяют отдельные функции в изоляции с известными входными данными и ожидаемыми результатами.

| Модуль ядра | Файл тестов | Что тестируется |
|-------------|-------------|-----------------|
| `core/data.py` | `tests/test_data.py` | Загрузка CSV/Excel/Parquet, SQL-запросы |
| `core/validate.py` | `tests/test_validate.py` | Вывод типов, проверка ограничений |
| `core/prepare.py` | `tests/test_prepare.py` | Парсинг дат, приведение числовых типов, очистка |
| `core/aggregate.py` | `tests/test_aggregate.py` | Результаты GroupBy, сводные таблицы, скользящие окна |
| `core/explore.py` | `tests/test_explore.py` | Статистики распределений, матрицы корреляций |
| `core/tests.py` | `tests/test_tests.py` | t-тест, Манна-Уитни, хи-квадрат, Bootstrap |
| `core/models.py` | `tests/test_models.py` | STL-декомпозиция, SARIMAX, обнаружение аномалий |
| `core/attribution.py` | `tests/test_attribution.py` | Аддитивная, мультипликативная атрибуция, Shapley Values |
| `core/simulation.py` | `tests/test_simulation.py` | Монте-Карло, сценарные переборы параметров |
| `core/triggers.py` | `tests/test_triggers.py` | Пороговые, отклоняющие и наклонные триггеры |
| `core/audit.py` | `tests/test_audit.py` | Добавление записей, получение лога, формат записей |
| `core/i18n.py` | `tests/test_i18n.py` | Ключи переводов, поведение при отсутствии ключа |
| `core/report.py` | `tests/test_report.py` | Рендеринг HTML, генерация PDF |
| `core/merge.py` | `tests/test_merge.py` | JOIN, UNION, конкатенация, диагностика pitfalls |

#### Ключевые области модульных тестов

- **Парсинг дат**: форматы ISO, ДД.ММ.ГГГГ, ММ/ДД/ГГГГ, неоднозначные даты, некорректные строки, обработка часовых поясов.
- **Парсинг чисел**: запятые в качестве десятичного разделителя («1.234,56»), символы валют, строки с процентами, научная нотация, пустые строки, обработка NaN.
- **Агрегация**: сумма, среднее, медиана, количество, минимум, максимум по сгруппированным данным; многоуровневая группировка; нулевые ключи групп.
- **Атрибуция**: проверка того, что вклады факторов в сумме дают общую дельту (аддитивная) или в произведении — отношение (мультипликативная).
- **Триггеры**: срабатывание порогового значения, отсутствие срабатывания, граничные случаи.
- **Валидация**: отсутствующие обязательные колонки, неверные типы, дублирующиеся строки, значения вне допустимого диапазона.

### 2. Интеграционные тесты

Интеграционные тесты проверяют сквозной конвейер с использованием образцов данных из директории `sample_data/`.

```
Загрузка CSV -> Валидация схемы -> Очистка колонок -> Агрегация -> Запуск тестов -> Экспорт
```

Ключевые интеграционные сценарии:

- **Слияние нескольких файлов**: загрузить два CSV, объединить по ключевой колонке, проверить форму результата.
- **Полный конвейер**: импортировать сырые данные, применить приведение типов, отфильтровать строки, сгруппировать, запустить t-тест, сгенерировать PDF-отчёт.
- **SQL round-trip**: загрузить из CSV, записать в in-memory SQLite, прочитать обратно, сравнить DataFrames.
- **Генерация отчёта**: выполнить анализ, создать HTML-отчёт, проверить наличие всех разделов.

---

### 3. Золотые тесты (Golden Tests)

Золотые тесты сравнивают текущий вывод с сохранёнными эталонными файлами для обнаружения регрессий.

- Эталонные файлы хранятся в `tests/golden/`.
- Каждый золотой тест загружает фиксированный датасет, выполняет анализ и сравнивает результат с сохранённым эталоном (CSV или JSON).
- При намеренном изменении поведения обновляйте эталонный файл командой `pytest --update-golden`.

Примеры золотых тестов:

| Тест | Датасет | Что проверяется |
|------|---------|-----------------|
| STL-декомпозиция | monthly_sales.csv | Трендовая и сезонная компоненты |
| Атрибуция (аддитивная) | multivariate.csv | Таблица вкладов факторов |
| Bootstrap ДИ | ab_test.csv | Границы доверительного интервала |

---

### 4. Диагностические предупреждения

Тесты проверяют, что система выдаёт соответствующие предупреждения в граничных случаях:

| Условие | Ожидаемое предупреждение |
|---------|--------------------------|
| Менее 5 наблюдений | «Слишком мало наблюдений для надёжного теста» |
| Нулевая дисперсия в колонке | «Колонка имеет нулевую дисперсию» |
| Отсутствует обязательная колонка | «Обязательная колонка не найдена: {name}» |
| NaN в метрической колонке | «Метрика содержит {n} пропущенных значений» |
| Даты вне допустимого диапазона | «Обнаружены даты вне ожидаемого диапазона» |
| Группа с одним наблюдением | «Группа {name} содержит только 1 строку» |
| Высокая мультиколлинеарность (VIF > 10) | «Обнаружена мультиколлинеарность среди факторов» |

## Запуск тестов

### Полный набор тестов

```bash
python -m pytest tests/ -v
```

### С отчётом о покрытии

```bash
python -m pytest tests/ -v --cov=core --cov-report=term-missing
```

### Быстрая проверка работоспособности

```bash
python smoke_test.py
```

Smoke-тест загружает образцы данных, запускает каждую функцию модулей ядра и проверяет отсутствие исключений. Выполняется менее чем за 10 секунд.

### Запуск конкретного файла тестов

```bash
python -m pytest tests/test_tests.py -v
```

### Запуск тестов по шаблону имени

```bash
python -m pytest tests/ -v -k "attribution"
```

## Рабочий процесс непрерывного тестирования

1. Написать или изменить код в `core/`.
2. Запустить соответствующий файл тестов: `pytest tests/test_<модуль>.py -v`.
3. Запустить полный набор перед коммитом: `pytest tests/ -v`.
4. Убедиться, что покрытие не снизилось: `pytest --cov=core`.
5. Запустить `smoke_test.py` в качестве финальной проверки.
